server/failover： Disaster recovery of the server through IP
URL： https://{host}/api/server/failover
HTTP request method： PUT
Support Format： JSON
Disaster recovery request parameter description：
Parameter Required or not Participate sign or not Type Description
Key Yes Yes String DNS company authorizes the key, the interface invokes the credentials, can be generated the acquisition in the profile

Example of disaster recovery request parameters： 
                            
                            {
                                "key": "eb34c94e105e4c6663ae633be4269c20",
                                "ip": "121.80.12.117",
                                "nonce": "b05c18ff317440bb5c8c30c65aa660f3",
                                "timestamp": 1517622267,
                                "sign": "ed37bfe77097e765bce495458496db3d9f38b2a0"
                            }
Sign rule：
1：All parameters except "sign" are sorted in ascending order by key, then extreme DNS company Secret(secret=xxxxxx) and the sorted parameter (key=value) are spliced ​​with &, the secret does not participate in sorting, place in the last one.
Example: "ip=121.80.12.117&key=eb34c94e105e4c6663ae633be4269c20&nonce=b05c18ff317440bb5c8c30c65aa660f3& timestamp=1517622267&secret=84e70c9e3c52429263f9607549443890"
2: Sort and combine the string to operate by sha1。
Example: "ed37bfe77097e765bce495458496db3d9f38b2a0"
Disaster recovery success return parameter： 
                                    
                                        "OK"
Disaster recovery fail return parameter： 
                                
                                    {
                                        "status_code": 400,
                                        "error_code": "401-02-001",
                                        "message": "Sign Error！"
                                    }
Disaster recovery fail return parameter description
Parameter Type Description
status_code number return the status code
error_code string error coding
message string error message
